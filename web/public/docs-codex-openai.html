Codex + OpenAI Compatible

如何把 Codex 接到 OpenAI 兼容协议

与 Claude Code 不同，当前这套 Codex CLI 可以直接通过 provider 配置连接 OpenAI 兼容接口。对于 Nerve Centers，推荐方式是把 ~/.codex/config.toml 里的 model_provider 指到一个 proxy provider，再把 base_url 设为站点提供的 https://robot2.indevs.in/v1。

推荐链路：Codex → Nerve Centers /v1 不需要 Anthropic 协议转换层 适合直接走 OpenAI 兼容 endpoint
查看 Claude Code 指南

核心差异

Claude Code

需要协议适配层

Claude Code 需要 Anthropic 风格入口，因此前面通常还要有一层网关做协议转换。

Codex

可以直连 OpenAI 兼容接口

这台机器上的 Codex CLI 已经在使用 provider 配置直连兼容接口，base URL 就是 Nerve Centers 的 /v1。

生产建议

单独建一个 provider

不要直接覆盖默认登录模式，给 Nerve Centers 单独配置一个 proxy provider 更稳妥，也方便以后切换。

前置条件

  1. 你已经拿到一个可用的 Nerve Centers 令牌。
  2. 你的站点可通过 https://robot2.indevs.in/v1 提供 OpenAI 兼容接口。
  3. 本机已安装 Codex CLI。
说明： 本页示例按当前机器上实际可运行的 Codex 配置格式编写，而不是假想字段。

步骤 1：配置 Codex provider

在 ~/.codex/config.toml 中添加或修改如下配置： 

model_provider = "proxy"
model = "gpt-5.4"
model_reasoning_effort = "high"
personality = "pragmatic"

[model_providers.proxy]
name = "Nerve Centers"
base_url = "https://robot2.indevs.in/v1"
env_key = "NEWAPI"

这里的关键点有两个：

  • model_provider = "proxy" 表示让 Codex 走自定义 provider。
  • base_url 直接指向 Nerve Centers 的 OpenAI 兼容根路径，而不是主页域名。

步骤 2：注入 API Key

当前配置里 env_key = "NEWAPI"，因此需要在启动 Codex 的终端里提供对应环境变量： 

# macOS / Linux
export NEWAPI=sk-your-nerve-centers-token

# Windows PowerShell
$env:NEWAPI="sk-your-nerve-centers-token"
不要混淆： 这里不是给业务代码用的 `OPENAI_API_KEY`，而是给 Codex provider 自己读取的环境变量名；它由 env_key 决定。

步骤 3：启动 Codex

配置好之后，直接在目标仓库里运行：

codex

# 或非交互模式
codex exec "检查当前项目的 README 并给出接入建议"

如果你看到 Codex 正常进入会话，而不是报认证错误或找不到 provider， 说明基础接入已经通了。

验证点

检查 1

Codex 能正常启动

启动后应能进入交互界面或执行 `codex exec`，而不是在初始化阶段就失败。

检查 2

Nerve Centers 有请求记录

在 Nerve Centers 侧应能看到对应模型调用、日志或计费记录。

检查 3

模型名可用

如果 provider 已通但模型名无效，Codex 会在真正发请求时失败，这时需要回到 model 字段调整为站点可用模型。

常见问题

  • 把 base_url 写成主页地址而不是 /v1 根路径。
  • env_key 和实际环境变量名不一致，导致 Codex 读不到令牌。
  • 把 Claude Code 的 Anthropic 网关方案照搬到 Codex，结果多加了一层不必要的转换。
  • 模型名填成站点未开放或未映射的模型，表现为连接通了但请求失败。

推荐做法

  1. 给 Nerve Centers 单独保留一个 proxy provider。
  2. 把 base_url 固定成 https://robot2.indevs.in/v1。
  3. 用独立环境变量保存令牌，例如当前页示例里的 NEWAPI。
  4. 优先先用一个稳定模型验证连通，再逐步切到你真正要用的模型。
一句话总结： Codex 接 Nerve Centers 的关键不是协议翻译，而是把 ~/.codex/config.toml 的自定义 provider 配到正确的 /v1 endpoint 和令牌变量上。
