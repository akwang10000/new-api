Codex + OpenAI 兼容接口

如何把 Codex 接到 OpenAI 兼容接口

和 Claude Code 不同，Codex CLI 可以直接通过 provider 配置连接 OpenAI 兼容接口。对 Nerve Centers 来说，推荐做法是给 Codex 单独配一个 provider，再把 base URL 指向本站提供的 `https://robot2.indevs.in/v1`。

核心差异

Claude Code

中间需要一层中转

Claude Code 不能直接连接这里的 OpenAI 兼容接口，通常需要先经过一层中转，把请求转换成可用格式。

Codex

可以直接连 OpenAI 兼容接口

Codex CLI 支持用 provider 方式直接连接 OpenAI 兼容接口，因此只需要把 base URL 和令牌配置好。

建议做法

单独建一个 provider

不要覆盖默认配置。给 Nerve Centers 单独建一个 provider，后续切模型或切服务更稳妥。

前置条件

  1. 你已经拿到一个可用的 Nerve Centers 令牌。
  2. 站点可通过 `https://robot2.indevs.in/v1` 提供 OpenAI 兼容接口。
  3. 本机已经安装 Codex CLI。
说明： 本页示例按当前站点真实可用的接入方式编写，不使用占位用的假地址。

步骤 1：配置 Codex provider

在 `~/.codex/config.toml` 中新增或修改下面这段配置： 

model_provider = "proxy"
model = "gpt-5.4"
model_reasoning_effort = "high"
personality = "pragmatic"

[model_providers.proxy]
name = "Nerve Centers"
base_url = "https://robot2.indevs.in/v1"
env_key = "NEWAPI"
  • `model_provider = "proxy"` 表示让 Codex 使用自定义 provider。
  • `base_url` 要直接指向 Nerve Centers 的 `/v1` 根路径，而不是首页域名。

步骤 2：注入 API Key

当前配置里使用 `env_key = "NEWAPI"`，所以启动 Codex 的终端里要提供同名环境变量： 

# macOS / Linux
export NEWAPI=sk-your-nerve-centers-token

# Windows PowerShell
$env:NEWAPI="sk-your-nerve-centers-token"
不要混淆： 这里不是必须写成 `OPENAI_API_KEY`，而是由 `env_key` 决定具体读取哪个环境变量名。

步骤 3：启动 Codex

配置完成后，直接在目标仓库里运行 Codex 即可： 

codex

# 或非交互模式
codex exec "检查当前项目的 README 并给出接入建议"

如果 Codex 能正常进入会话，而不是立刻报认证错误或找不到 provider，说明基础接入已经打通。

验证点

检查 1

Codex 能正常启动

启动后应能进入交互界面，或者正常执行 `codex exec`。

检查 2

Nerve Centers 有请求记录

在 Nerve Centers 后台应能看到对应模型调用、日志或计费记录。

检查 3

模型名称可用

如果 provider 已通但模型名无效，请求会在真正发起时失败，此时需要改回站点已开放的模型名。

常见问题

  • 把 `base_url` 写成首页地址，而不是带 `/v1` 的接口根路径。
  • `env_key` 和实际环境变量名不一致，导致 Codex 读不到令牌。
  • 把 Claude Code 那套“先过一层翻译器”的方案照搬给 Codex，结果多加了一层不必要的中转。
  • 模型名称填写成站点未开放或未映射的模型，表现为连通了但请求失败。

推荐做法

  1. 给 Nerve Centers 单独保留一个 `proxy` provider。
  2. 把 `base_url` 固定写成 `https://robot2.indevs.in/v1`。
  3. 用独立环境变量保存令牌，例如当前示例中的 `NEWAPI`。
  4. 先用一个稳定模型验证连通，再切到你实际要用的模型。
一句话总结： Codex 接入 Nerve Centers 的关键不是协议转换，而是把 `~/.codex/config.toml` 中的 provider、`/v1` 地址和令牌变量配置正确。
