Claude Code + 中转层

如何让 Claude Code 用上 Nerve Centers

先说结论：Claude Code 不能像 Codex 一样直接连接 Nerve Centers 的 OpenAI 兼容接口。更通俗地说，Claude Code 说的是另一种“接口语言”，所以中间要先加一层“翻译器”，再把请求转到 Nerve Centers。

一句话理解

Claude Code

不能直接连

Claude Code 不能直接把请求发到 Nerve Centers 的 OpenAI 兼容接口，否则大概率会卡在请求格式不匹配这一步。

中转层

负责翻译

中转层的工作就是把 Claude Code 发出的请求，转换成 Nerve Centers 能识别和接收的请求。

Nerve Centers

继续做统一入口

现有的令牌、模型映射、计费和日志能力都可以继续复用，不需要把现有网关推翻重来。

为什么不能直接连接

最容易理解的方式不是去谈底层协议，而是把它看成“双方说的不是同一种话”：

  • Claude Code 默认期待的是它自己那套接口格式。
  • Nerve Centers 当前对外提供的是 OpenAI 兼容接口。
  • 所以 Claude Code 直接去连 Nerve Centers，经常表现为域名能通，但请求本身过不去。
可以把它想成翻译问题： Claude Code 先把话说给中转层，中转层再把内容翻译成 Nerve Centers 能接收的格式。

推荐接法

正确链路不是强行直连，而是先加一层中转服务，然后再接到现有网关： 

Claude Code
  -> Middleware
  -> Nerve Centers /v1
  • Claude Code 只连接中转层。
  • 中转层再去连接 Nerve Centers。
  • Nerve Centers 继续负责模型调用、令牌校验、计费和日志。

你需要准备什么

  1. 一个可用的 Nerve Centers 令牌。
  2. 一个可访问的 OpenAI 兼容地址，例如 `https://robot2.indevs.in/v1`。
  3. 一层放在 Claude Code 和 Nerve Centers 中间的中转服务。
重点不是替换 Nerve Centers： 只是给 Claude Code 前面补上一层中转，让它能把请求顺利送进来。

怎么判断已经接对

检查 1

Claude Code 能正常启动

启动时不再卡在认证或请求格式错误，而是可以正常进入工作流。

检查 2

中转层有请求记录

说明 Claude Code 已经成功把请求打到了中间层，而不是直接连错地方。

检查 3

Nerve Centers 有真实调用

如果中转层配置正确，你会在 Nerve Centers 侧看到真实的模型调用记录。

常见误区

  • 把 Claude Code 直接指向 `https://robot2.indevs.in/v1`。
  • 以为模型名对上了就能用，实际接口格式不匹配也不行。
  • 把 Codex 的直连方案原样照搬到 Claude Code。
