Claude Code + Gateway

如何让 Claude Code 接入 routeropenai

按 Anthropic 官方文档，Claude Code 可对接的不是任意 OpenAI 兼容地址，而是满足 gateway contract 的接口形态。routeropenai 对外主入口是 OpenAI 兼容 `/v1`，所以 Claude Code 要接入它，通常需要一层对外说 Anthropic Messages、对内再转发到 routeropenai `/v1` 的 gateway。

一分钟结论

不能直连

Claude Code 不是 OpenAI-compatible client

把 Claude Code 直接指向 `https://routeropenai.xyz/v1`，常见问题不是域名不通，而是它请求的接口契约就不一样。

官方要求

Gateway contract 有明确边界

Anthropic 官方要求 gateway 至少暴露一种受支持的接口形态，其中最常见的是 Anthropic Messages：`/v1/messages` 与 `/v1/messages/count_tokens`。

routeropenai

上游聚合能力仍可复用

模型路由、供应商聚合和上游访问仍可放在 routeropenai；只是 Claude Code 前面需要一层说 Anthropic 协议的 gateway。

为什么不能直接连

更准确的说法不是“Claude Code 需要随便一个中转层”，而是“Claude Code 需要一个满足官方 gateway contract 的接口层”：

  • Anthropic 官方支持的 gateway 形态包括 Anthropic Messages、Bedrock InvokeModel 或 Vertex rawPredict。
  • 如果走 Anthropic Messages 形态，至少要提供 `/v1/messages` 与 `/v1/messages/count_tokens`，并转发 `anthropic-beta`、`anthropic-version` 这些头。
  • OpenAI 兼容 `/v1/chat/completions` 不在 Claude Code 官方支持的直接接入列表里，所以不能把 Claude Code 当作普通 OpenAI 客户端来配。
结论： Claude Code 接入 OpenAI 兼容上游时，准确做法不是“直接改个 base_url”，而是先放一层满足 Claude Code contract 的 gateway，再由它去调用 routeropenai 的 OpenAI 兼容接口。

推荐接法

生产可落地的链路是：Claude Code 对 gateway 说 Anthropic 协议，gateway 再去调用 routeropenai 的 OpenAI 兼容 `/v1`：

Claude Code
  -> Claude-compatible gateway (/v1/messages, /v1/messages/count_tokens)
  -> routeropenai /v1
  • Claude Code 通过 `ANTHROPIC_BASE_URL` 指向 gateway，而不是 `https://routeropenai.xyz/v1`。
  • gateway 对外暴露 Claude Code 需要的接口形态，对内把请求翻译并转发给 routeropenai。
  • routeropenai 继续负责上游模型聚合、渠道配置与实际模型调用。

你需要准备什么

  1. 一个可用的 routeropenai 令牌，供 gateway 访问 `https://routeropenai.xyz/v1`。
  2. 一个满足 Claude Code 官方要求的 gateway，例如能提供 Anthropic Messages 端点的代理层。
  3. Claude Code 中把 `ANTHROPIC_BASE_URL` 指向这个 gateway，并提供 `ANTHROPIC_AUTH_TOKEN` 或等效认证方式。
不要写成直连 OpenAI 兼容地址： 如果你的层只会暴露 `/v1/chat/completions` 这类 OpenAI 兼容路径，它对 Codex 没问题，但对 Claude Code 仍然不够。

Claude Code 侧典型配置

下面只是方向示例：Claude Code 指向 gateway，gateway 再指向 routeropenai `/v1`。不要把 `ANTHROPIC_BASE_URL` 直接写成 `https://routeropenai.xyz/v1`。

# Claude Code side
export ANTHROPIC_BASE_URL=https://your-gateway.example.com
export ANTHROPIC_AUTH_TOKEN=your-gateway-token

# Gateway side
UPSTREAM_OPENAI_BASE_URL=https://routeropenai.xyz/v1
UPSTREAM_OPENAI_API_KEY=sk-your-routeropenai-token

怎么判断已经接对

检查 1

Claude Code 能正常启动

启动时不再卡在协议不兼容、认证失败或 request-format 错误，而是能进入正常工作流。

检查 2

gateway 能看到 `/v1/messages` 请求

这说明 Claude Code 已经命中了符合 contract 的中间层，而不是误打到 OpenAI 兼容入口。

检查 3

routeropenai 有真实上游调用

如果 gateway 配置正确，你会在 routeropenai 一侧看到真实模型调用记录。

常见误区

  • 把 Claude Code 直接指向 `https://routeropenai.xyz/v1`。
  • 只做了 `/v1/messages`，却漏掉 `/v1/messages/count_tokens` 或漏传 `anthropic-*` 头，导致部分功能失效。
  • 把 Codex 的 OpenAI 直连方案原样照搬到 Claude Code。
本页只讲一个核心点：Claude Code 连接 OpenAI 兼容上游时，需要的是满足 Anthropic 官方 contract 的 gateway，而不是把 OpenAI 兼容地址直接填进去。
