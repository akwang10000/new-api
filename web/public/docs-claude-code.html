Claude Code + 中转层

如何让 Claude Code 用上 Nerve Centers

先说结论：Claude Code 不能像 Codex 一样直接连接 Nerve Centers 的 OpenAI 兼容接口。更好理解的说法是，Claude Code 说的是另一种接口语言，所以中间要先加一层“翻译器”。

一分钟结论

不能直连

不是地址通了就算接通

Claude Code 直接指向现有 OpenAI 兼容地址时，常见问题不是域名不通，而是请求格式对不上。

中转层

它的职责很明确

中转层只负责把 Claude Code 发出的请求，改写成 Nerve Centers 能识别和接收的格式。

现有网关

原有能力继续保留

模型映射、令牌校验、日志和计费能力仍然放在 Nerve Centers，不需要重做一套。

为什么不能直接连

这件事最容易理解的方式，不是去谈底层协议细节，而是把它看成“双方说的不是同一种话”：

  • Claude Code 默认期待的是它自己的那套接口格式。
  • Nerve Centers 当前对外提供的是 OpenAI 兼容接口。
  • 所以 Claude Code 直接去连 Nerve Centers，通常会表现成地址能访问，但请求无法正确处理。
可以把它理解成翻译问题： Claude Code 先把请求发给中转层，中转层再把它翻译成 Nerve Centers 能处理的格式。

推荐接法

正确做法不是强行直连，而是先放一层中转服务，再把请求导向现有网关： 

Claude Code
  -> 中转层
  -> Nerve Centers /v1
  • Claude Code 只连接中转层。
  • 中转层再去连接 Nerve Centers。
  • Nerve Centers 继续负责模型调用、权限校验、计费和日志。

你需要准备什么

  1. 一个可用的 Nerve Centers 令牌。
  2. 一个可访问的 OpenAI 兼容地址，例如 `https://robot2.indevs.in/v1`。
  3. 一层放在 Claude Code 和 Nerve Centers 中间的中转服务。
重点不是替换 Nerve Centers： 只是给 Claude Code 前面补一层中转，让它能顺利把请求送进来。

怎么判断已经接对

检查 1

Claude Code 能正常启动

启动时不再卡在认证错误或请求格式错误，而是能正常进入工作流。

检查 2

中转层能看到请求

这说明 Claude Code 已经命中了中间层，而不是直接打到了错误地址。

检查 3

Nerve Centers 有真实调用

如果中转层配置正确，你会在 Nerve Centers 一侧看到实际模型调用记录。

常见误区

  • 把 Claude Code 直接指向 `https://robot2.indevs.in/v1`。
  • 以为模型名对上了就能用，实际上接口格式不匹配也不行。
  • 把 Codex 的直连方案原样照搬到 Claude Code。
本页只解决一个核心问题：Claude Code 接入 Nerve Centers 时，为什么中间需要一层中转服务。
